Какие сущности должны быть в проекте?
Сейчас я буду смотреть аналоги и попробую сделать на основе них какой то бэк
По сути это будет очередной таск трекер но также будут сущности которые будут отвечать за навыки 
и накопление в них уровня

Какие по итогу сущности? 
1. Задача - содержит id, название, описание, дату создания, 
дедлайн её выполнения, статус задачи, приоритет, сложность, повторение (для задач привычек),
награда за выполнение, список навыков которые задача прокачивает, штраф за просрочку - ГОТОВО

2. Характеристика (у пользователя будет несколько изначальных характеристик, также он сможет добавлять свои) 
- содержит id, название, описание, уровень прокачки, тек опыт, необходимый опыт до следующего уровня,
 список навыков влияющих на прокачку
 
3. Профиль пользователя - содержит его личные данные, список характеристик, навыки,
оценку прокачки персонажа на основании текущих данных


Календарь - непосредственно сам календарь, 
список задач на текущий день, список просроченных задач, список законченных задач 
- это страница, а не сущность!

4. Достижение - содержит id, название, условия для получения, статус - получено/не получено, 
награда, которая содержит в себе - получаемый опыт, прокачиваемую характеристику

5. Навык - id, название, описание, уровень, тек опыт, опыт необходимы до след уровня, прокачиваемые характеристики  


Пункты меню - Задачи, Статус, Календарь, Достижения, Статистика
Боковое меню - Профиль, Настройки, Навыки
